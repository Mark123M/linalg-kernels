# CLAUDE.md

## Goal

Build an efficient **batched real symmetric eigendecomposition** (`eigh`) for the **B200 GPU** in `eigh.py`, submitted through the popcorn evaluation infrastructure. Among passing submissions, ranking is by runtime (geometric mean over benchmark cases). Batched `512 x 512` is the central target — the first kernel is optimized for N ≈ 512; `1024`, `2048`, `4096` cover larger square factors.

Solution plan:
1. Blocked Householder tridiagonalization (**done**: panel kernel + `tridiagonalize_` driver in `eigh.py`, D/E outputs, verified on B200; perf tuning continues)
2. Divide and conquer eigensolver (**done**: `tridiag_eig_dc_` — htev leaves + native prep kernel + bmm merges; see "D&C tridiagonal eigensolver" section)
3. Backtransform for eigenvectors (**done**: reverse compact-WY T builder + batched cuBLAS applications, connected through `full_eigh_out_` and the production `custom_kernel`; official 13-case B200 gates all pass)

## Problem spec (condensed from AGENTS.md)

- Input: `A`, `batch x n x n` CUDA tensor, `torch.float32`, symmetric up to FP32 roundoff.
- Output `(Q, L)` in the `torch.linalg.eigh` convention: `Q` orthonormal eigenvector columns (FP32), `L` ascending eigenvalues (FP32).
- Correctness is **residual-gated in FP64**, not elementwise: eigen-equation `A@Q - Q@diag(L)`, reconstruction `Q@diag(L)@Q.T - A`, orthogonality `Q.T@Q - I`, each relative to the matrix L1 norm. Signs/rotations within eigenspaces are free. Internal low-precision (FP16/FP8/NVFP4) is allowed; returned factors must be FP32 and meaningful.
- Stress cases: rank-deficient, near-rank-deficient, repeated/clustered eigenvalues, diagonal, banded, row-scaled, mixed-conditioning batches, LAPACK DDRVST-inspired types. Robustness is ranked, not only gated.
- Benchmark shapes (batch × n): 20×32, 40×176, 40×352, **640×512**, 60×1024, 8×2048, plus mixed/rankdef/clustered/nearrank/lapack variants at 512 and 1024.

## Current kernel state (`eigh.py`)

Panel tridiagonalization kernel, one CTA per batch matrix, `num_threads` (128/256) split into row groups of `threads_per_row` (N-dependent, 8→256; constructor arg `threads_per_row=` overrides, used to exercise `warps_per_row>1` paths at small N locally). Implements the full DLATRD column flow steps 1-5 (only the trailing DSYR2K remains). Per panel column `i` (loop `cutlass.range(panel_size, unroll_full=True)`):

1. **Column load + refresh**: 1D async tiled copy of the panel subcolumn into smem `sCol`, predicated `col_idx >= i and col_idx < panel_n` (cp.async zero-fills masked lanes, so `v` is zero below `i` for free). While the cp.async is in flight, the column refresh (dlatrd.f:297) accumulates `corr[p] = Σ_{j<i} V[p,j]·W[i-1,j] + W[p,j]·V[i-1,j]` in registers (**coefficient row = the diagonal row of column i, v-space `i-1`** — dlatrd.f's GEMVs read `W(I,1)`/`A(I,1)`, local row I; an earlier version read v-space row `i`, the subdiagonal — self-consistent with the then-mirror and invisible to every kernel==mirror gate, but not a similarity transform; caught 2026-07-13 by the independent gold Householder anchor, eig error 2.5e-1 → 5e-8 after the fix): **thread-per-row, serial-j** (thread owns `p = tidx + m·num_threads`, the w-append striding, `tiler_n/num_threads` accumulators) — unlike the outer pass's warp-per-row, register accumulation bridges the cp.async wait with no smem staging, and at fixed `j` consecutive threads read consecutive `p` (coalesced; the per-`j` coefficients `W[i,j]`/`V[i,j]` are warp-broadcast reads). No reductions inside, `j`-trip CTA-uniform. After the wait+barrier, a masked subtract `sCol[p] -= acc[m]` (`i <= p < panel_n`: preserves v's zero prefix and the zero pad tail) + **one new barrier** publishes the refreshed column; `alpha` and everything downstream read it unchanged. Uniform at i=0 (zero-trip j-loop, subtract of 0). Rejected alternative (register-only refresh, no store-back): the norm needs the `sColBcast` fragment layout and w-formation re-reads `sCol` after the matvec clobbered `sB`; keeping corr alive costs +tiler_n smem (`sCorr` — breaks N=4096 local launch) for a ~4-iteration smem-pass saving. Revisit as a B200 micro-opt.
2. **Reflector (every row group redundantly)**: `sColBcast` = broadcast view of `sCol` with layout `(rows_per_tile, tiler_n) stride (0,1)`, partitioned by the 2D matvec tiled copy — so each row group's register copy of the column lines up elementwise with its A-tile rows. `norm_sq` via `row_sum` (warp reduction, `block_sum` through `reduction_buffer` when `warps_per_row > 1`). Semantics (LAPACK `dlarfg` convention): `x` includes alpha at index `i`; `beta = -sign(alpha)*||x||`; `tau = (beta-alpha)/beta`; `v` tail `= x/(alpha-beta)` with `v[i] = 1` forced via a dynamic-`if` element loop. Zero-tail columns take the `tau=2` sign-flip reflector (valid under residual gating) instead of LAPACK's `xnorm==0 => H=I` shortcut — deliberate, per input restrictions.
3. **Inner correction GEMVs** (dlatrd.f:316-318, 322-324): `s_w = Wᵀv`, `s_v = Vᵀv` into smem arrays `sSw/sSv`, placed between the tile-0 prefetch and the matvec loop to hide the first gmem fetch. Row group `r` owns panel column `j = jt*rows_per_tile + r` (a row of the gmem workspace), so fragments align with `tVrV` like `sColBcast`. `j >= i` is masked (never branched — a group-divergent branch around `row_sum` would break `block_sum`'s CTA barrier); dynamic trip count `ceil(i/rows_per_tile)` is CTA-uniform.
4. **Pipelined GEMV** `b = A' @ v` (the memory-bound bottleneck): K-stage cp.async pipeline over row tiles, `K = self.stage` (constructor/`compile`/`tridiagonalize_` arg `stage`), smem ring `sA (rows_per_tile, tiler_n, K)`. Prologue issues the first `K-1` tiles **before the inner GEMVs** (so those loads stream while the corrections run and the matvec opens at steady state); the loop then issues tile `t+K-1`, holds `cp_async_wait_group(K-1)` (K-1 groups in flight), and reduces tile `t` from buffer `t % K` with `row_sum(a * v)`; lane 0 of each row group stores into smem `sB`. `K=2` reproduces the old double buffer bit-for-bit. Column bound via static `predicate_k(limit=panel_n)`; row bound via scalar `if` (quack convention). No barrier around `sA` — each thread reads back exactly what it copied (the buffer it overwrites at iter `t` was consumed at `t-1`). `K=2` reproduces the old double buffer bit-for-bit. **Default `stage=2`.** Local RTX 4050 640×512 ps=8 showed K≥6 ≈ 2× faster than K=2, but that did NOT transfer — **B200 nsys (b640_n512_ps32, K=6 confirmed via 53,520 B dynamic smem) shows the panel kernel is occupancy-bound, not DRAM-bandwidth-bound**: DRAM read ~25% avg / 55% at the peak panel / 72% max, compute-warps-in-flight ~13% avg (25% max), SM issue ~24%. It uses 80 regs/thread + 128 threads + smem, so occupancy is CTA-residency-capped: K=2 (~20 KB) → 6 CTAs/SM (register-limited); K=6 (53.5 KB) → 4 CTAs/SM (smem-limited) → **52 ms vs 44 ms at K=2, an ~18% regression**. The RTX 4050 win was a low-bandwidth artifact (~190 GB/s saturates; B200 has ~40× and touches ¼). **B200 stage sweep DONE (`--tri --stages`, 640×512 ps=32 cublas): K=2 46.4 ms, K=3 47.4, K=4 47.5, K=6 53.2 — monotonically worse, so prefetch depth is a DEAD END here; K=2 is optimal and is the default.** Do not re-explore stages. **Register-cap/occupancy is ALSO a DEAD END for 640×512 — and the CTA-residency-ceiling framing above was measuring the wrong ceiling.** The `min_blocks_per_mp` knob (kernel `stage`-style arg; `nvvm.minctasm`, launch-bounds min-blocks; also passes exact smem so the carveout sizes the partition — see `Eigh._smem_bytes`, validated against the 102480 B launch error; default 0 = ptxas-chosen regs, byte-identical) forces a register cap for more CTAs/SM. **B200 sweep DONE (`--tri --min-ctas 0,6,7,8`, 640×512 ps=16 cublas): mc0 44.95 ms, mc6 45.08 (neutral — isolates the carveout overhead, negligible), mc7 45.16, mc8 45.92 — monotonically worse (pure spill cost, 80→73→64 regs), no upside.** Reason (the correct wave analysis): 640 matrices = 640 CTAs on 148 SMs → **640/148 = 4.32 CTAs/SM, a single wave with every SM busy, already *below* the 6/SM register ceiling** (capacity 6×148=888 ≥ 640). So occupancy here is *work-limited*, not capacity-limited — raising the ceiling to 8 fills nothing (there aren't enough CTAs), and the 13% warps-in-flight is **intra-CTA** stalling, not too-few-CTAs. The lever for 640×512 is per-CTA stall reduction (**ncu stall-reason breakdown**: long-scoreboard/memory vs short-scoreboard/barrier → the latter means the serial 32-column recurrence / 32×-per-panel trailing-block re-read needs restructuring), NOT occupancy. **Separately, the small-batch large-N shapes are CTA-COUNT-starved, not occupancy-starved: 60×1024 = 60 CTAs (88 SMs idle), 8×2048 = 8 CTAs (140 SMs idle — the ~400 ms outlier). Their lever is *creating more CTAs* — split each matrix across multiple CTAs — which no per-SM occupancy/register knob can address.** **IMPLEMENTED (2026-07-14) as the cluster-cooperative panel mode — see "Cluster-cooperative panel" below. B200 8×2048 ps=16 tridiag: cs=1 391.7 ms → cs=8 83.6 → cs=16 76.7 (5.1×, vs torch_tridiag 164.2 = 2.1×); 4×2000 (tail panel): 397.1 → 75.1 → 52.1, gates green everywhere.**
5. **Outer correction GEMVs** (dlatrd.f:319-321, 325-327): `b -= V·s_w + W·s_v`, reusing the **column refresh's** thread-per-row / serial-`j` register accumulation — thread owns rows `p = tidx + m·num_threads`, loops `j < i` (no mask), reading `V[j,p]`/`W[j,p]` coalesced (consecutive threads → consecutive `p`) with the `s_w[j]`/`s_v[j]` coefficients as smem broadcast reads, then a masked `sB[p] -= acc[m]` (shares the `acc` registers with the refresh). Replaced the earlier warp-per-row/warp-shuffle variant, whose lane-per-`j` reads were strided by `tiler_n` across the row-major workspace. Skipped via a `if i > 0` guard (no reduction/barrier inside; each thread reads back the `sB` rows it writes).
6. **w formation** (dlatrd.f:328-331): `w = τ·b + (-τ/2)(wᵀv)·v` using `wᵀv = τ·(bᵀv)` (one `row_sum` over a broadcast `sB` view); a thread-strided loop appends `v`/`w` as rows of the gmem workspace.

**Outputs are LAPACK/cuSOLVER-style D (B×N) / E (B×(N-1)), both FP32** — `mTri` is gone. `E[col] = beta`: one scalar store by thread 0 right after the reflector, hidden under the tile-0 prefetch + inner GEMVs. `D[col]`: the diagonal of column `i` sits at v-index `p = i-1` (never loaded into `sCol`, so the zero prefix/norm are untouched), and its refresh correction is exactly the thread-per-row refresh's `acc[m]` on the owning thread — the apply loop adds `if p == i-1: mD[col] = A[col,col] - acc[m]` (one scalar gmem load+store on one thread). At `i=0` the diagonal was finalized by the *previous* panel's rank-2k (trailing block starts exactly at panel_start), so it's a raw copy issued while the column cp.async is in flight. `D[N-1]` is gathered by the driver after the last (1×1-trailing) update. **Backtransform outputs (2026-07-14)**: `Tau[col] = tau` is stored by thread 0 next to the `E` store ((B, N-1) FP32; drivers take an optional `tau=` kwarg and allocate when omitted — compiled signature always includes it), and the w-append loop persists `mCol[p] = vp` into `data` column `col` (vp already carries sCol's zero prefix, so rows panel_start+1..col get explicit zeros — including the diagonal at p=i-1, read for D before the append — the subdiagonal gets the explicit unit, and the tail follows). **The panel kernel therefore mutates `data`**: safe within the factorization (every later matvec read of an overwritten column is multiplied by a zero v coefficient; later panels/rank-2k touch only trailing blocks — traced exhaustively), but callers that reread `data` must clone first (`custom_kernel` and the runner's `factor_panel` now do).

**Cluster-cooperative panel (`cluster_size`, 2026-07-14)**: `Eigh(cluster_size=cs)` fans each matrix's panel across a thread-block cluster of `cs` CTAs (sm90+ only; `MAX_CLUSTER_SIZE=16`; launch `grid=[cs, batch, 1]`, `cluster=[cs,1,1]`, block x = rank, y = matrix). Design (iket-informed: matvec = 97% of column time at n=2048, all serial phases ≈ 15 µs): the **matvec row tiles split into contiguous per-CTA slabs** (`tiles_per_cta = ceil(num_tiles/cs)`, panel-fixed so each CTA re-reads only its slab every column; empty slabs on late/tail panels just run the barriers); the **serial phases run redundantly in every CTA** (column load + refresh, reflector, inner corr — identical inputs and reduction order ⇒ bit-identical tau/v per CTA, loads L2-hot); **outer corr, w-formation, V/W/data appends are slab-local** (outer-corr loads predicated to the slab keep non-slab sB rows at zero, which makes the full-width `bᵀv` row_sum a per-slab partial for free). Cross-CTA traffic per column: each rank publishes its partial `bᵀv` to a gmem slot (`Btv`, (batch, 16) FP32 workspace, `panel_btv` in `FullEighWorkspace`), then two cluster barriers (`fence_acq_rel_gpu` + `cluster_arrive`/`cluster_wait`): one after the publish (all CTAs re-sum the cs slots in fixed order — deterministic, graph-replay-safe, no atomics), one at column end (orders every rank's gmem appends before the next column's redundant refresh; subsumes the old CTA barrier). D/E/Tau/printf stores gate on rank 0. **cs=1 traces exactly the single-CTA kernel** (verified bit-identical on D/E/tau/V/W/data across N∈{512,67,200} incl. tail panels and threads_per_row=64) and launches without a cluster. Kernel/`__call__` signature is now `(data, D, E, Tau, V, W, Btv, panel_start)` (Btv passed even at cs=1 — uniform ABI); compile keys gained `cluster_size`. Production dispatch: `dispatch_cluster_size(batch, n, device)` — `EIGH_CLUSTER_SIZE` env override, else cs=16 for n≥2048, else 1; falls back to 1 pre-sm90. Harness: `--cluster-size` (local runner + modal `--tri`/`--nsys`/`--trace`), `--clusters 1,8,16` sweep (modal `--tri`, gates re-run per width), `--cluster-size` on `--official-eigh` sets the env override (0 = production dispatch).

**`panel_start` is a runtime kernel argument** (dynamic Int32; `panel_n`, `num_tiles`, the `predicate_k` limit, and all `domain_offset` bases derive from it dynamically) — one compile per (N, ps) serves every panel of a full factorization. Layouts/smem/workspace are sized once for the widest panel (`tiler_n = ceil(N-1, num_threads)`); dynamic bounds mask the shrinkage. Measured cost of losing the static predicate constant-folding on B200: none (panel ps=16 unchanged, see status). Kernel/`__call__` signature: `(data, D, E, Tau, V, W, panel_start)`. Latent tail-panel bug (pre-existing, found 2026-07-14): when the tail's padded workspace rows < the full panel's rows (e.g. N=200 ps=32: tail 7 pads to 16 < 32), the driver's leading-rows slice is non-contiguous and the cuBLAS rank-2k binding raises "must be contiguous" — no benchmark shape hits it (ps=16 tails all pad to exactly 16), fix before shipping other panel sizes.

**V/W panels live in gmem workspace**, not smem (removes the smem ceiling — every benchmark N/ps launches; a smem-resident variant is a possible later optimization for N≤1024). Per-batch shape `(rows, cols)` from `Eigh.workspace_shape()`: rows = panel_size padded to `rows_per_tile`, cols = `tiler_n`; row-major so inner-GEMV reads and w-appends are coalesced (outer reads are strided but L2-hot). **Callers must `torch.zeros`-allocate**: unwritten rows/pad columns are read under masks before first write and must be finite (`0 * NaN` poisons reductions); the kernel writes only valid data, so the invariant survives graph replays (later panels leave stale-but-finite tails, which the same masks make harmless).

**Full tridiagonalization driver `tridiagonalize_(data, D, E, v_ws, w_ws, panel_size=, backend=, queue_handle=)`** (product code, submitted): the N-1 reflector columns split into `(N-1)//ps` full panels + one `(N-1)%ps` tail panel (second jit-cached compile; workspace passed as leading-rows slices to match its fake-tensor shape); each panel launch is followed by its rank-2k at `panel_start` (`rank2k_update_` now takes `panel_start`, not `k` — the tail start is not a multiple of ps; the last update's trailing block is 1×1, finalizing `A[N-1,N-1]` for the final `D[:, -1]` gather). `data` is mutated — callers pass a work copy. Everything is on the ambient queue and CUDA-graph capturable (the Python loop unrolls into the graph).

Barriers per column: inside reductions only when `warps_per_row > 1` (`reduction_buffer` WAR); unconditional after the refresh subtract (thread-strided `sCol` rewrite, read cross-thread by alpha + the reflector fragments), after the matvec (publishes `sB`+`sSw`/`sSv` to the outer pass), after the outer pass (threads rewrote their `sB` rows, read cross-thread by the w-formation `w^T v` reduction), and at column end (`sCol` WAR vs next cp.async; also orders this CTA's gmem V/W appends — all its threads share one SM's L1).

Post-panel DLATRD update: `rank2k_update_` mutates the unreduced trailing block with `A -= V Wᵀ + W Vᵀ` at the dsytrd.f DSYR2K offsets — trailing block starts at global row/col `panel_start + ps`, i.e. workspace column `p = ps - 1` (an earlier `+1` variant skipped the first trailing row/col — exactly the column the next panel's first reflector reads — and passed its own self-consistent gate; the gate now encodes the LAPACK offsets). The `cublas` backend issues two `CUBLAS_COMPUTE_32F` strided-batched GEMMs without transposing the row-major workspace (bit-identical to pedantic locally; pedantic is a debug mode that forbids B200 fast paths). The `cublasdx` backend uses a 32x32x`panel_size` block GEMM, reuses one register accumulator for both products, and performs one predicated in-register epilogue — one C read-modify-write vs the cuBLAS path's two. `run_panel_with_update` orders the panel and update on the current stream. Both native paths remain embedded in `eigh.py` for Popcorn's single-file submission format; cuBLAS links from the torch wheel's bundled libs (cu13 or per-library cu12/cu13 layouts, `-lcublas` fallback) and MathDx headers resolve via `MATHDX_ROOT`, pip `nvidia-mathdx` under site-packages, or CPATH.

Panel sizes: any compile-time size in `[1, 128]` is accepted when it fits in `N`. Sizes through 32 retain full unrolling; larger sizes use `unroll=4` to avoid code-size/compile-time blowup. cuBLASDx modules compile lazily per `(panel_size, architecture)` and use `MATHDX_ROOT` or a system-visible MathDx/CUTLASS include pair; no repository-local/vendored fallback is used.

## Fused 20×32 HEEV specialization

`custom_kernel` now dispatches exact `(batch, N) == (20, 32)` inputs to a separate native cuSolverDx HEEV extension. One CTA handles one matrix and performs dense tridiagonalization, HTEV, and eigenvector formation inside a single device launch; the general panel/D&C/backtransform pipeline and its workspaces are not entered. The extension allocates and returns Q/L from its C++ binding and reuses a cached 20-entry status buffer.

Each CTA loads the input once into shared memory and computes the maximum absolute row sum `scale`. It solves `A / scale + 2I` (`scale=1` for the zero matrix), then returns `(lambda-2)*scale`; this keeps the internal spectrum in `[1,3]`, avoids the known zero-spectrum HTEV failure, preserves the input, and retains FP32 output. `EIGH_SMALL_HEEV_NT={32,64,128,256,512,1024}` and `EIGH_SMALL_HEEV_FILL={lower,upper}` are cache-keyed sweep controls.

**B200 sweep (2026-07-14, JSON `profiles/eigh_small_heev/small_heev_20260714T214258Z.json`)**: all six width/triangle configurations passed 11 stress families with zero HEEV statuses, including zero/rank-deficient/clustered and high/low-magnitude inputs. The best observed configuration, now the production default, was **NT=128/upper: 0.147886 ms custom vs 0.153777 ms torch = 1.040×**. This is a **2.11× reduction** from the stable pre-fusion 0.313 ms custom baseline and is worth about a 5.9% aggregate-geomean improvement by itself. The torch control varied 0.148–0.159 ms across separately timed configurations, so the 4% head-to-head edge is not yet a robust win; the harness now alternates provider order per repeat for confirmation. NT=128 was directionally better than NT=32 (0.152–0.148 vs 0.157 ms), while the upper-vs-lower difference was negligible except in the final NT=128 run.

## D&C tridiagonal eigensolver (`tridiag_eig_dc_` in `eigh.py`)

Batched divide & conquer (LAPACK dstedc/dlaed0-4 chain) for step 2, fully on-device and CUDA-graph capturable — the deflation count K never reaches the host. Motivating profile (2026-07-14 nsys, `profiles/eigh_nsys/b640_n512_ps16_cublas_*`): raw cuSolverDx htev at N=512 silently fell to `Thread()` execution (one thread solves one 512×512 problem serially, 1 MB gmem eigenvector matrix per thread) and took **38 s** vs 43 ms tridiag. Block-mode htev with `Job<all_vectors>` is smem-resident by design: `4N² ≲ smem` caps it at **N ≤ ~240 on B200, ~157 on sm89** — `_load_htev_module` now **raises** on the thread-mode fallback instead of compiling the ~1000x-slower path.

Structure (one compile total for the prep module; htev compiles per leaf size):
- **Tree**: `_dc_tree(n, leaf)` halves until ≤ leaf (default 32, B200-tuned — see leaf sweep below; ≥3 required; ≤ 2-ish distinct leaf sizes). Tears subtract `|E[cut]|` from both straddling diagonals up front (dlaed0.f:263-268), rho read back from `E` per merge. No data-dependent natural splits — a negligible `E` at a tear just yields tiny z → full deflation, so the tree shape is uniform across the batch (graph-friendly).
- **Leaves**: batched block-mode htev over `B × leaves` subproblems per distinct size (gather D/E slices → one launch → scatter eigenvalues into `D`, vectors into the Z ping buffer's diagonal blocks).
- **Merge = one m×m `S_eff` + two bmm's.** The native `dc_prep_kernel` (plain-CUDA `load_inline` module `eigh_dc_prep_v1`, no fast-math — the secular iteration needs exact division; one CTA of 256 threads per (matrix, merge), grid `(G, B)`) folds *everything column-side* into `S_eff`: dlaed2 deflation (rule A tiny-z, rule B close-eigenvalues → Givens recorded then applied to S rows in reverse), the dlaed4 secular solve (one lane per root; **fixed-weight two-pole quadratic in the eta-step matched to (W, DW)** — dlaed4.f:633-658 interior, :324-340 last root — with Newton + bracket-midpoint safeguards, 40 fp32 iterations; value-only frozen-rest models 2-cycle on mid-cluster roots, and bisection-only loses relative accuracy near poles: both were observed failing before porting the LAPACK step), the Löwner ẑ recompute (dlaed3.f:260-290 — this is what keeps eigenvectors orthogonal under clustering), deflated pass-through unit columns, and the final ascending sort (column order of S_eff). Then per merge: `Z_out[top] = Q1 @ S[:n1, :]`, `Z_out[bot] = Q2 @ S[n1:, :]` — plain `torch.bmm` into strided `out=` views (verified no-copy), ping-ponged between the caller's `Z` and `ws.z_alt` (parity chosen so the last level lands in `Z`). **Unbalanced trees**: levels that don't cover all rows copy the pass-through segments' diagonal blocks across the ping buffers (missing this gave orth=1.0 at N=65 — balanced N=512/1000 hid it).
- Smem: `16·m` bytes/CTA (survivor poles, weights→ẑ, roots, origins); `m ≤ 4096` (attribute set at `prepare_dc`). Scratch: `fbuf (B,4,N)` fp32 + `ibuf (B,6,N)` int32, per-merge slices disjoint by construction. Workspace via `dc_workspace(batch, n, device, leaf_size)`; `warm_dc(n, leaf)` before graph capture.
- API: `tridiag_eig_dc_(D, E, Z, ws, *, leaf_size=32, queue_handle=0)` — `D` in/out (diag → ascending eigenvalues), `E` read-only, `Z (B,N,N)` gets eigenvector *columns*. fp32 throughout (matches slaed*; fp64 not needed — see gates).

Verified (2026-07-14, local sm89): fp64+fp32 Python mirror (scratchpad `test_dc.py`) vs `torch.linalg.eigh` on 6 stress families (random, diagonal, clustered/repeated, rank-deficient half-zero spectrum, graded 1e-12, zero-tear) at N ≤ 1000 — fp64 resid/orth ~1e-15, fp32 ~1e-6. Native (`test_dc_native.py`): same families, N ∈ {65, 128, 176, 256, 512, 1000}, leaf ∈ {16, 32, 128} — resid/orth ≤ 1e-5, eig ≤ 6e-6, all OK. Graph capture (`test_dc_graph.py`): replays bit-identical, equal to eager. ## CuTe DSL performance rules (measured in this project)

- The DSL is a **tracing JIT**: Python runs at compile time; layouts/partitions/`local_tile`/`domain_offset` with `const_expr` args cost **nothing at runtime**. Dynamic *values* (predicates, limits, pointer offsets) compile to a few integer ops — cheap, like hand-written CUDA indexing.
- **Never respecialize per loop iteration** (`cutlass.range_constexpr` + per-i `const_expr` tile sizes): it duplicates the kernel body per iteration — measured 22x compile time and up to 1.85x runtime regression (I-cache) at panel_size=32, while ceil-rounding made the "smaller" tiles identical anyway. Use `cutlass.range(..., unroll_full=True)` with one traced body + dynamic predicates instead.
- **Keep predicate limits static when possible**: with a static limit, most of `predicate_k`'s per-thread booleans constant-fold away. Making the bound depend on runtime `i` turned them into live registers and measured 3–11% *slower* — for <2% traffic saved. Columns `< i` are already nullified by `v`'s zeros.
- `cp.async` needs ≥4-byte transactions: `num_copy_elems=1` copies only work for fp32.
- Smem budget: `sCol + sB + K*rows_per_tile*tiler_n` elements (V/W are gmem; `K = stage`, the matvec pipeline depth). At N=512 each `sA` stage is 8 KB, so K=6 ≈ 52 KB; at N=4096 each stage is 32 KB, so K=6 ≈ 224 KB (fits B200's 227 KB but is the practical ceiling there). Exceeding the arch limit fails at **launch**, not compile ("launch shared memory exceeds current GPU arch"), with the allocated byte count in the message (local sm_89 ~99 KB ceiling ⇒ K≤11 at N=512, K≤2 at N=4096). Higher K also lowers CTAs/SM (occupancy), which on B200 is the actual limiter — see step 4; the default `stage=2` is deliberately low, not smem-driven.
- Don't add synchronization unless needed for correctness; document why each barrier exists.
- Use `cutlass.Constexpr` / `const_expr` type annotations aggressively so the compiler sees static structure.
- The DSL warns at ≥64 iterations of a static loop; treat that as a hard smell.
- **Never use a ternary expression with a dynamic condition** (`x if dyn_cond else y`): the condition is resolved at trace time and one branch is silently baked in (trace-time-arbitrary — can even differ between compiles). Use a dynamic `if` statement, which the preprocessor lowers correctly. Cost of a masked scalar update this way is negligible.
